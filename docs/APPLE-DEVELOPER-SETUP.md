# Apple Developer — подготовка к покупке и настройке

Цель: получить платный Apple Developer аккаунт, чтобы:
1. Подписывать приложение для дистрибуции (TestFlight / Ad Hoc / App Store).
2. Использовать **Network Extension** (системный VPN) — без него приложение
   остаётся SOCKS-only и требует внешний клиент для маршрутизации.
3. Сделать установку максимально юзабельной (TestFlight вместо SideStore).

---

## 1. Что именно покупать

**Apple Developer Program — Individual** — **$99/год**.

- Организация (Organization, $99/год) требует D-U-N-S номер и юр.лицо.
  Для нас **Individual проще и быстрее** (паспорт, без юр.лица).
- **Enterprise Program ($299/год)** — НЕ подходит: это для внутренних
  корпоративных приложений, не для публичной раздачи. Apple банит за
  раздачу публичных приложений через Enterprise-сертификаты.

### Что нужно для регистрации
- [ ] **Apple ID** с включённой двухфакторной аутентификацией (2FA).
- [ ] **Способ оплаты**: карта, принимаемая Apple (часто проблема из РФ —
      см. раздел «Оплата из РФ» ниже).
- [ ] **Юридическое имя** латиницей, совпадающее с данными карты/паспорта.
- [ ] Возраст 18+.
- [ ] Приложение Apple Developer на iPhone (для подтверждения личности
      может потребоваться) ИЛИ регистрация через developer.apple.com.

### Оплата из РФ (важно)
Apple не принимает карты МИР / российских банков. Рабочие варианты:
- [ ] Иностранная карта (Казахстан, Армения, Грузия, Турция, ОАЭ и т.п.).
- [ ] Виртуальная карта зарубежного банка/финтеха.
- [ ] Покупка через посредника (риск блокировки аккаунта — нежелательно
      для продукта, который должен жить долго).

> Рекомендация: завести **отдельный новый Apple ID** под этот аккаунт,
> не привязанный к личному, и платить иностранной картой. Так личный
> Apple ID не пострадает при любых проблемах.

## 2. Что завести ПОСЛЕ покупки (App Store Connect + Certificates)

### 2.1. App ID (идентификатор приложения)
- [ ] Bundle ID основного приложения: `org.prismovpn.app`
      (или ваш выбранный reverse-domain; должен быть уникальным).
- [ ] Bundle ID расширения Network Extension:
      `org.prismovpn.app.tunnel`
      (NE-расширение всегда имеет суффикс к основному bundle id).

### 2.2. Capabilities (включить для App ID)
- [ ] **Network Extensions** → Packet Tunnel Provider.
- [ ] **App Groups** (для обмена данными между app и NE-расширением):
      `group.org.prismovpn.app`.
- [ ] **Keychain Sharing** (опционально — хранить токен/ключи безопасно).

> ⚠️ Capability **Network Extensions / Packet Tunnel** доступна ТОЛЬКО на
> платном аккаунте. На бесплатном (Personal Team) её включить нельзя —
> поэтому системный VPN сейчас и не работает.

### 2.3. Profiles & Certificates
- [ ] Development certificate (для отладки на устройстве).
- [ ] Distribution certificate (для TestFlight/App Store).
- [ ] Provisioning profiles для обоих bundle id (app + tunnel),
      с включёнными capabilities.

### 2.4. App Store Connect
- [ ] Создать приложение (имя, иконка, описание).
- [ ] Настроить **TestFlight** — это лучший способ раздачи бета-версии
      (до 10 000 внешних тестеров, ставится в один тап, без SideStore).

## 3. Дистрибуция: варианты после покупки

| Способ | Кол-во юзеров | Установка | Модерация Apple | Подходит нам |
|---|---|---|---|---|
| **TestFlight** | до 10 000 | 1 тап | Лёгкая (review бета) | ✅ ДА (старт) |
| App Store | ∞ | 1 тап | Жёсткая (VPN-правила) | ⚠️ риск отказа |
| Ad Hoc | до 100 UDID | вручную UDID | нет | для теста |
| SideStore (текущее) | ∞ | сложно, перезаводить 7 дней | нет | временно |

> **План:** сразу после покупки → TestFlight. Это решает «сделать
> юзабельным»: юзер ставит из TestFlight в один тап, без возни с SideStore.
> App Store — отдельный заход позже (там жёсткая модерация VPN-приложений,
> нужно скрыть упоминания обхода блокировок — см. ребрендинг).

## 4. App Store review — риски для VPN (на будущее)

Если пойдём в публичный App Store:
- [ ] Приложение НЕ должно явно рекламировать «обход блокировок РКН /
      государственной цензуры» в описании — Apple отклоняет.
- [ ] Позиционировать как «privacy / secure DNS / personal VPN».
- [ ] Нужна Privacy Policy (URL) и понятное описание сбора данных.
- [ ] Категория: Utilities или Productivity.
- [ ] Никаких строк `Prismo`, `PrismoVPN`, `Prismo` в UI/логах —
      см. задачу ребрендинга.

## 5. Что я (агент) подготовлю в коде заранее (до покупки)

Чтобы после покупки «всё включилось» минимальными правками:
- [ ] Зафиксировать `PRODUCT_BUNDLE_IDENTIFIER` = `org.prismovpn.app`
      в `project.yml`.
- [ ] Завести target Network Extension (заглушка `PacketTunnelProvider`),
      App Group, entitlements-файлы — но НЕ включать обязательную сборку,
      пока нет сертификатов (чтобы CI не падал).
- [ ] Убрать все palящиеся строки (Prismo/Prismo) — независимо от аккаунта.
- [ ] Автоимпорт подписки по токену + валидация — независимо от аккаунта.

> После покупки: вписать Team ID в `project.yml`, включить capabilities,
> сгенерировать профили — и собрать подписанный билд для TestFlight.

## 6. Статус ребрендинга (вырезание Prismo/Prismo)

### ✅ Уже вырезано (runtime-утечки — логи, UI, сеть)
- Лог-строки: `"Starting Prismo tunnel"` → `"Starting tunnel"`.
- Ошибки движка: `PrismoVPN`/`Prismo` → нейтральные («Core engine…»,
  «Failed to start the tunnel»). Обновлены и `.strings` (en/ru) и call-sites.
- **Wire User-Agent** `Prismo/1` → `Mozilla/5.0` (это уходило в сеть!).
- TOML-комментарии, runtime-папка `Prismo` → `Prismo`.
- GCD-очереди `io.prismo.*` → `org.prismovpn.*` (видны в крэш-логах).
- Share-ссылки теперь `prismodns://` (старые `prismo://` ещё принимаются).
- SOCKS-дефолты `prismo` → `prismo`.
- Внутренний тип `PrismoEngine` → `TunnelEngine` (+ файл переименован),
  helper `prismoPlainInput` → `prismoPlainInput`.

### ⚠️ Осталось (рисковый build-рефактор — НУЖНО РЕШЕНИЕ)
Эти имена видны только при декомпиляции бинарника (symbol table), НЕ в
логах/UI/сети. Переименование задевает систему сборки:
- Имена модуля/таргета: `PrismoKit`, `PrismoApp`, `PrismoApple` →
  `PrismoKit`, `PrismoApp`, `PrismoApple`.
- Файлы: `Package.swift`, `project.yml`, `.xcodeproj`, схемы, папки
  `Sources/PrismoKit`, `Sources/PrismoApp`, `Tests/PrismoKitTests`.
- Авто-имя ресурс-бандла `PrismoApple_PrismoKit.bundle` в
  `AppLocalization.swift` (SwiftPM собирает как `{Package}_{Target}.bundle`).
- **Внешняя зависимость**: `ResolverCatalog.rawBaseURL` указывает на
  `https://hub.mos.ru/pete9/prismo/...` — это репозиторий АВТОРА апстрима.
  Нужно перенести списки резолверов на НАШУ инфраструктуру (иначе и утечка
  имени, и зависимость от чужого репо, который могут удалить).

> Рекомендация: модуль-рефактор делать одним заходом и проверять сборкой на
> macOS/CI (на Windows Swift-сборку не проверить). Резолвер-URL — отдельно,
> после того как разместим списки у себя.
