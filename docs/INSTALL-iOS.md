# Prismo DNS для iPhone — установка

> **Когда это нужно.** Prismo DNS — это DNS-туннель: он пробивает интернет там,
> где заблокировано почти всё (жёсткие блокировки операторов, «социальные»
> тарифы, активные блокировки РКН). Он не для скорости, а для связи там, где
> обычный VPN уже не проходит. Для повседневного использования берите обычную
> VLESS-подписку в Happ.

На iPhone приложение ставится **не из App Store** (Apple не пускает туда
DNS-туннели). Ставим через **SideStore** — это безопасно (приложение
подписывается вашим собственным Apple ID, как в TestFlight) и бесплатно.

Компьютер нужен **один раз** на ~20 минут для настройки. Дальше всё обновляется
на самом телефоне.

---

## Что понадобится

- iPhone на iOS 16+
- Бесплатный **Apple ID**
- Компьютер (Windows/Mac) — только для первичной настройки SideStore
- USB-кабель
- Ваш **токен Prismo** (команда `/token` в боте)

---

## Шаг 1. Установить SideStore (один раз, с компьютера)

1. Скачайте **iLoader** (бесплатный, открытый): https://iloader.site
2. Подключите iPhone к компьютеру по USB, разрешите «Доверять».
3. В iLoader войдите своим Apple ID, выберите устройство → **Install SideStore**.
4. На iPhone: **Настройки → Основные → VPN и управление устройством** →
   выберите профиль с вашим Apple ID → **Доверять**.
5. **Настройки → Конфиденциальность и безопасность → Режим разработчика** →
   включите (телефон перезагрузится).
6. Установите из App Store бесплатный **LocalDevVPN**, откройте, нажмите
   **Connect** (он нужен SideStore, чтобы ставить и обновлять приложения).
7. Откройте **SideStore**, войдите тем же Apple ID, в **My Apps** нажмите на
   счётчик «7 DAYS» у SideStore, чтобы завершить настройку.

Подробная официальная инструкция со скриншотами: https://docs.sidestore.io/docs/installation/install

---

## Шаг 2. Установить Prismo DNS (.ipa)

1. Скачайте свежий **PrismoDNS-unsigned.ipa**:
   - на странице релизов нашего репозитория, **или**
   - из артефактов последней успешной сборки CI (Actions → Build iOS → артефакт
     `PrismoDNS-unsigned-ipa`).
2. Откройте **SideStore → My Apps → «+»** в левом верхнем углу.
3. Выберите скачанный `PrismoDNS-unsigned.ipa`.
4. Дождитесь установки. Иконка Prismo DNS появится на экране «Домой».

> Раз в 7 дней приложение нужно **обновлять** (подпись бесплатного Apple ID
> живёт 7 дней). В SideStore: **My Apps → Refresh All**. С включённым
> LocalDevVPN и фоновым обновлением это происходит почти автоматически.

---

## Шаг 3. Подключиться по токену (в одно касание)

1. В Telegram-боте отправьте `/token`, скопируйте токен.
2. Откройте **Prismo DNS** → **«Подключить по токену»**.
3. Вставьте токен → **Импортировать**.
   Приложение само заберёт сервер и ключ — вручную ничего вводить не нужно.
4. Нажмите кнопку питания (Connect). Статус станет **Подключено**, появится
   локальный прокси `SOCKS5 127.0.0.1:41080`.

> Можно ещё короче: если у вас есть ссылка вида
> `prismodns://token?value=ВАШ_ТОКЕН`, просто откройте её — профиль
> импортируется автоматически.

---

## Шаг 4. Направить трафик в туннель

Prismo DNS поднимает локальный **SOCKS5 на `127.0.0.1:41080`**. Чтобы весь
трафик iPhone шёл через него, используйте прокси-клиент (например **Happ** или
Shadowrocket):

- Тип прокси: **SOCKS5**
- Адрес: `127.0.0.1`
- Порт: `41080`

После этого включите подключение в прокси-клиенте — интернет пойдёт через
DNS-туннель Prismo.

---

## Если что-то не работает

| Симптом | Решение |
|---|---|
| Приложение перестало открываться | Прошло 7 дней — обновите в SideStore (**Refresh All**) |
| «Этот токен недействителен» | Проверьте, что подписка активна; получите токен заново через `/token` |
| «Подписка истекла или неактивна» | Продлите подписку в боте |
| Туннель долго не поднимается | Выключите другие VPN-приложения и перезапустите Prismo DNS |
| Медленно | Это нормально для DNS-туннеля. Для скорости — обычная VLESS-подписка в Happ |

---

## Почему не из App Store?

DNS-туннель работает в фоне способом, который Apple не разрешает в App Store,
поэтому единственный путь на iOS — сайдлоад через SideStore. Подпись делается
**вашим личным Apple ID**, никакие чужие сертификаты не используются — это
устойчиво к блокировкам Apple, в отличие от «enterprise»-сборок.
